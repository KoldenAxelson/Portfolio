# Neofolio docs

Project documentation. Start here if you're forking and want to make it
yours, or if you're a contributor looking for the why behind a decision.

The README at the repo root is the GitHub front door — quickstart, feature
list, license. These docs go deeper.

## Map

| Doc | What it covers |
|---|---|
| [architecture.md](./architecture.md) | File structure, data flow, component map, how the layout system works |
| [customization.md](./customization.md) | Make it yours — identity, colors, fonts, nav |
| [content.md](./content.md) | Authoring projects, articles, network contacts, archive entries (via `make` or by hand) |
| [deploying.md](./deploying.md) | GitHub Pages + Cloudflare Pages walkthroughs, env vars, base paths |
| [performance.md](./performance.md) | Lighthouse targets, what's optimized, how to verify |
| [seo.md](./seo.md) | AI-readability strategy, JSON-LD, directory submissions, rel=me, syndication |
| [design-philosophy.md](./design-philosophy.md) | Why we made specific choices |
| [roadmap.md](./roadmap.md) | What's done, what's planned, deferred decisions — read this first if you're picking up the project |

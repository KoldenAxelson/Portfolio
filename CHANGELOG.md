First public release.

Neofolio is an opinionated Astro 5 + Vue 3 + Tailwind portfolio template
designed for two audiences simultaneously: humans browsing and LLMs
summarizing. Every load-bearing decision â€” static-first builds, the
JSON-LD entity graph, structured content collections â€” flows from that

### Added

**Pages.** Long-scroll homepage with sticky sidebar + scroll-spy +
per-element typewriter; `/projects` and `/projects/[slug]` (Tier 1 case
studies); `/articles` and `/articles/[slug]`; `/cv`; `/certificates`;
`/uses`; `/now`; `/network`; `/404`. Plus machine-readable endpoints
`/rss.xml`, `/feed.json`, `/llms.txt`, `/robots.txt`,
`/sitemap-index.xml`, `/humans.txt`.

**Content collections.** Zod-validated schemas for `projects`, `posts`,
`archive`, `network`, and `certificates`. Drafts are excluded at build
time. Interactive scaffolds via `make project`, `make post`,
`make contact`, `make archive`.

**JSON-LD entity graph.** Site-wide `WebSite` and `Person` carry stable
`@id`s. Per-page schemas (`BlogPosting`, `SoftwareApplication` /
`SoftwareSourceCode` / `CreativeWork`, `EducationalOccupationalCredential`,
`ItemList`, `Person.hasOccupation`, `Person.hasCredential`) all reference
the canonical entities via `@id` so parsers merge them into one graph.
Base-path-aware: deploys under sub-paths (GitHub Pages project sites)
emit `@id`s that match the actually-served URLs.

**Agent-driven setup.** `AGENTS.md` for orientation and hard invariants;
`SETUP_AGENT.md` for an 11-phase runbook a Claude / Cowork / Code agent
follows after `make setup`. `TODO_AI.md` tracks deferred AI-readability
work. Two helper scripts back the flow: `scripts/clean-examples.mjs`
(wipes every `example-*` file in one call) and `scripts/verify-setup.mjs`
(post-build placeholder + JSON-LD + sitemap sanity check). Wired into
`make clean-examples` and `make verify`.

**ATS-friendly `/cv`.** Candidate name as the prominent H1; visible
contact line (location, `mailto:` email, optional `tel:` phone, plain-text
GitHub / LinkedIn URLs); month-precision date ranges derived from the ISO
`startDate` / `endDate` fields in `src/data/cv.ts` (`Jan 2024 â€“ Present`);
`Technologies:` keyword on each role; paragraph-shaped Skills section
that legacy ATS parsers handle reliably.

**Default OG image** at `public/og-default.png` (1200Ă—630, dark-mode
palette, briefcase accent). Articles and projects override per-page via
their `cover:` frontmatter â€” both `og:image` / `twitter:image` meta and
the JSON-LD `image` field pick it up automatically.

**Cache headers.** `public/_headers` ships `immutable, max-age=1y` for
content-hashed `_astro/*` assets, fonts, textures, badges, covers; HTML
gets short-browser / longer-CDN with `stale-while-revalidate`. Cloudflare
and Netlify pick it up automatically; GitHub Pages ignores it and uses
its own CDN defaults. Addresses the Lighthouse "use efficient cache
lifetimes" audit.

**Deploy targets.** GitHub Pages workflow (auto-detects `BASE_PATH` from
repo name; falls back to `/REPO_NAME` for project sites). Cloudflare
Pages workflow ships disabled (`workflow_dispatch` only) â€” flip to
`push` when ready. Both produce the same `dist/`.

**Theming.** Five CSS variables in `src/styles/global.css` cover light
and dark mode. Matte noise textures regenerated by `make noise`.
Self-hosted Inter font with `font-display: swap` (no third-party
render-block).

**Performance & accessibility.** Static HTML on first paint; Vue islands
hydrate `client:visible` (project filter) or `client:load` (network panel)
only. Lighthouse targets â‰Ą 90 / 100 / 100 / 100 documented and verified.
AA contrast in both modes; underlined links by default for color-blind
users; touch targets â‰Ą 36Ă—36; `aria-label` / `aria-current` /
`prefers-reduced-motion` honored throughout.

**LLM-readability.** `/llms.txt` generated at build time from `SITE` +
content collections (llmstxt.org spec). Permissive `robots.txt` with
explicit allow rules for GPTBot, ClaudeBot, anthropic-ai,
Google-Extended, PerplexityBot, CCBot. h-card microformat on the
homepage sidebar. `rel="me"` on outbound social links.

### Notes

- The `posts` collection name is intentional while the public URL is
  `/articles`. Renaming the folder would break existing slugs for
  forkers tracking the upstream.
- Cloudflare Pages Functions (`functions/api/contact.ts`) are opt-in. The
  static build works on any host; the `mailto:` fallback covers the
  contact form on GitHub Pages.
- The template ships with example content under `src/content/*` and an
  example identity in `src/config.ts`. A first task on every fork is
  `node scripts/clean-examples.mjs` followed by editing `src/config.ts`
  and `src/data/cv.ts` â€” `SETUP_AGENT.md` walks through this in order.
